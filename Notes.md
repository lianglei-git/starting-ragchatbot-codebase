
From Zayne's notes on this project:
这是一个还不错让你了解RAG系统是如何工作的项目，你可以用它来回答关于课程材料的问题。该项目使用了语义搜索和AI生成的响应。
这里使用的是 chroma 作为rag的核心工具，而该项目则是结合文档数据以及ai来完成在课程内部的回答。及引用。

你可以直接在vscode 中使用 .vscode.example 中的 launch.json 来调试项目。当然，你需要重新把launch.json 中的路径换到 .vscode目录下

RAG的流程：
**传统RAG：** 
文档 → 分块 → 向量化 → 存储到向量数据库
> 文档（你的文档） → 分块（自己做切分，如按段落、按标题、按作者等你认为关键的信息） → 向量化（从文本到向量数据的表示） → 存储到向量数据库（如ChromaDB）
👇
用户问题 → 向量化 → 向量相似度搜索 → 检索相关文档片段
👇
检索结果 + 用户问题 → 构建提示词 → 大模型生成 → 最终答案

该项目RAG流程：


用户问题 -> 发送给Claude,利用提供的tools工具结构化分析和输出 -> 根据结果进行，向量检索 -> 检索结果 + 用户问题 -> 构建提示词 -> Claude AI生成 -> 最终答案

> 从结果来看，先利用AI进行语义分类，这确保了检索结果的准确性，然后再进行向量搜索。


智能体RAG （Agentic RAG）：
用户问题 → AI决策 → [需要检索？] → 是 → 智能参数提取 → 执行检索 → 整合生成
                     ↓
                     否 → 直接回答

也就是代表，本项目实现的就是一个简单的智能体RAG。
